# Projektregeln für Codex

Sprache: Deutsch.

Projekt: Unity 2D IT-AA Lern-/Portfolio-Projekt.

Regeln:
- Immer vollständige Dateien liefern, keine Snippets.
- Jede C#-Datei bekommt einen Kopfkommentar mit Zweck, Verantwortung, Abhängigkeiten und Verwendung.
- Namespace-Struktur beibehalten: ITAA.*
- Eine Klasse pro Datei.
- Keine unnötigen Architektur-Overengineering-Lösungen.
- Bestehende Unity-Inspector-Referenzen möglichst nicht brechen.
- Szenen, Prefabs und Animator-Controller nur vorsichtig ändern.
- Bei Änderungen an UI, SaveSystem, Player oder Arthur immer Pfade angeben.
- Ziel: saubere Feature-driven Architecture, SOLID, wartbare Klassen.
- README.md immer aktualisieren
